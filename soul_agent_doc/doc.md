产品的定位：一款为用户随机生成小众·反向·放空式自驾旅游攻略建议的Agent产品
产品形态：做H5版本
计划阶段：浙江先行版


核心功能描述：用户在平台可以跟agent互动，为其随机选择一个距离近似300公里以内一个周边城市、县城作为目的地，生成一份偏向小众、主打精神放松、不密集打卡，体验当地生活、景色为主的自驾旅游攻略。生成一份邀请函类似的页面，邀请用户前往。

目标用户群体：青年、爱旅游、放松、不喜欢商业热闹、热爱自然的用户群体。


3.1 主流程：从"打开产品H5页面"到"上路"

以下是用户端完整主流程，每个节点标注了后台 Agent 的对应动作：

1.	用户打开 App → 进入"出发"页。Agent 静默加载：获取当前位置 / 当日天气 / 本周已去过的目的地（去重）。

2.	用户做三选：方向 + 方案类型 + 出发时间。系统记录，传入盲盒引擎。调用DeepSeek或者智谱的接口进行分析：
智谱提供搜索工具
每一个策划方案，首先使用智谱的模型API参考这个文档model.md提供的方案内容设计，




3.	盲盒旅游内容生成引擎执行随机抽取 → 候选城市池过滤 → 权重排序 → 吐出目的地 + 3 个地点。Agent 同步执行安全检查：路线、最近3天目的城市的极端天气等其他预警情况。

4.	Agent 生成"城市生活邀请函"：标题文案 + 3 个地点描述（烟火语气）+ 氛围指数 + BGM 推荐 + 封面图（MVP 阶段用预选图库，后期 AI 生成）。

整体内容按照小红书、Instagram的风格为主。
根据方向，有利好西行、利向东、利在北方、利南下的小标题，增加一种幸运感的描述。

5.	用户查看邀请函。可操作：接受 / 换个方向（Reroll，限 1 次）/ 收藏留存。

6.	出发：Agent 进入"旅行陪伴模式"——用户可随时发送文字/图片，Agent 给出情绪化回复或实时导览补充。

7.	到达打卡点：用户上传"非景点照片"，完成核销，积分入账。

8.	旅程结束：Agent 自动生成游记初稿（可编辑），用户一键分享至小红书/朋友圈。

3.2 异常与降级流程
•	目的地安全预警触发：Agent 不取消邀请函，而是在卡片内醒目标注风险项，由用户决策是否出发（体现"人机协同"原则）。

•	Reroll 耗尽：提示用户"你已经换过方向了，这次信任 Agent金泽一次 一次？"——心理锚定提升接受率。

•	网络中断：邀请函（本地的内容随时可见，可以保存为长图，分享时底部加一个固定的）本地缓存，离线可查看。

•	AI 生成内容质量低：设置 fallback 到预置优质文案库（人工运营维护约 50 条/城市）。
